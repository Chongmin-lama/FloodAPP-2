WHAT IS NOT DONE / KNOWN GAPS
------------------------------

MAP ACCURACY
  ⚠️  Map pins are placed at district centroids, not exact GPS coordinates.
      When an authority publishes an alert with a district selected, the pin
      appears at the center of that district — not the exact affected spot.
      To fix this properly, GPS coordinates would need to be captured when
      reporting (browser geolocation) and stored in the DB.

GUEST REPORTS ON MAP
  ⚠️  Reports submitted by guests (or logged-in citizens) do not appear on
      the map. Currently the map only shows official alerts published by
      authorities. Showing citizen reports as a separate pin layer was
      planned but not implemented.

REAL-TIME UPDATES
  ⚠️  The app uses manual fetch on page load. There is no auto-refresh,
      WebSocket, or polling. If someone submits a report while an authority
      is viewing the queue, they won't see it until they refresh.

EMAIL / SMS NOTIFICATIONS
  ⚠️  No notifications are sent when an alert is published or a report
      status changes. This was noted as a future feature.

PASSWORD RESET VIA EMAIL
  ⚠️  There is no "Forgot Password" flow. The only way to reset a password
      is for an admin to do it manually from the Users panel.

MOBILE NAV
  ⚠️  The sidebar is hidden on mobile (lg:block). There is no hamburger menu
      or mobile drawer for the dashboard navigation.

REPORT STATUS FOR GUEST SUBMISSIONS
  ⚠️  Guest reports (submitted without an account) cannot be tracked.
      The app correctly prompts guests to register after submitting, but
      there is no way to link a previously submitted guest report to a
      newly created account.

PAGINATION
  ⚠️  All tables load all records at once. If the database grows large,
      reports and user tables will become slow. No pagination implemented.


SUMMARY
-------
The application covers all the core requirements of a flood incident reporting
and community response platform. The three-role system (citizen, authority, admin)
is fully functional. The public can view live alerts and the map without logging in.
Citizens can report and track their incidents. Authorities can manage the full
response pipeline. Admins have complete control over users and content.

The gaps above are enhancements and polish items — none of them break the
core functionality of the app.
